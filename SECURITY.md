# Security Policy

This repository is a sanitized public-use blueprint.

## Public boundary

This repo may contain:

- structural descriptions
- placeholder files
- scaffold-generation instructions
- high-level governance concepts
- public documentation

This repo must not contain private implementation details.

## Do not add

- real application source
- private prompt bodies
- credentials
- provider keys
- runtime records
- audit logs
- save receipts
- generated hashes
- internal memory
- private configuration

## Contribution rule

Any future contribution should preserve the architecture while keeping implementation-specific material outside this public repository.
